Quiero que construyas un prototipo funcional frontend de una aplicación llamada FixHub.

Contexto del producto:
FixHub es una plataforma Managed Marketplace B2B para que empresas puedan contratar servicios técnicos y de mantenimiento brindados por PYMEs proveedoras. La plataforma debe estar orientada a clientes corporativos que necesitan buscar, comparar, contratar, pagar y seguir servicios como plomería, electricidad, gas, climatización y albañilería.

El objetivo es construir un MVP visualmente pulido, 100% mockeado, sin backend real. Todos los datos deben vivir en archivos TypeScript dentro del proyecto.

Stack obligatorio:
- React
- Vite
- TypeScript
- React Router DOM
- lucide-react para íconos
- CSS puro o CSS Modules, sin frameworks pesados
- No usar backend
- No usar APIs externas
- No usar base de datos
- No usar autenticación real

Diseño visual:
La app debe parecer una aplicación mobile first moderna, similar a las capturas de referencia de FixHub:
- Fondo principal blanco o gris muy claro
- Azul principal como color de marca
- Verde para estados exitosos o validaciones
- Tarjetas con bordes redondeados
- Sombras suaves
- Tipografía limpia y legible
- Diseño centrado en mobile, con ancho máximo aproximado de 430px
- Bottom navigation fija
- Header superior según pantalla
- Estados visuales claros: Programado, En camino, En progreso, Finalizado
- Proveedores con rating, reputación, ubicación, documentación validada y badges de confianza

Nombre de marca:
FixHub

Paleta sugerida:
- Azul principal: #0066FF o similar
- Azul oscuro: #003B7A
- Verde validación: #16A34A
- Amarillo rating: #FBBF24
- Gris fondo: #F6F8FB
- Gris texto secundario: #6B7280
- Blanco: #FFFFFF

Requisitos funcionales:
- La app debe navegar correctamente entre pantallas.
- No debe romper al refrescar rutas.
- El login debe redirigir a home.
- El pago debe simular la creación de un servicio activo.
- El servicio activo debe poder verse desde home.
- El seguimiento debe tener estados visuales.
- La calificación debe poder enviarse de manera simulada.
- Todo debe funcionar sin backend.

Criterios de calidad:
- Código claro y separado en componentes.
- Reutilizar componentes visuales.
- Mantener consistencia visual entre pantallas.
- Diseño mobile first.
- Evitar texto en inglés en la UI.
- No dejar pantallas vacías.
- No dejar TODOs visibles.
- No usar lorem ipsum.
- Usar datos realistas del proyecto FixHub.

Primero generá toda la estructura base, rutas, datos mockeados, estilos globales y componentes principales. Después implementá pantalla por pantalla hasta que la app sea navegable.
